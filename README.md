# Lab 1 — Telco Customer Churn (MLOps)

## Коротко про роботу
У лабораторній виконано:
- EDA (первинний аналіз, очищення даних)
- підготовку даних для моделі
- навчання `RandomForestClassifier`
- логування експериментів у **MLflow**
- порівняння моделей за різними `max_depth`

## Датасет
- **Telco Customer Churn**
- Задача: **бінарна класифікація** (`Churn`)
- Розмір: **7043 × 21**
- Пропуски: **11** (лише в `TotalCharges`)
- Дублікати рядків: **0**
- Дублікати `customerID`: **0**

## Що було реалізовано
- `train.py` з:
  - очищенням `TotalCharges`
  - `train_test_split` (зі `stratify`)
  - `ColumnTransformer` + `OneHotEncoder`
  - `RandomForestClassifier`
  - логуванням параметрів, метрик, моделі та артефактів у **MLflow**

## Проведені експерименти
- Baseline: `max_depth=None`
- Серія експериментів:
  - `max_depth = 2, 4, 6, 10, None`
- Додатково:
  - `min_samples_leaf` при `max_depth=6`

## Основні висновки
- Baseline-модель (`max_depth=None`) показала **перенавчання**:
  - дуже високі train-метрики
  - значно нижчі test-метрики
- Зменшення `max_depth` дозволило знизити overfitting.
- **Найкращий результат у серії `max_depth` показала модель з `max_depth=6`:**
  - `test_f1 = 0.624`
  - `test_recall = 0.791`
  - `test_roc_auc = 0.842`
- Для задачі churn більш інформативні метрики:
  - **F1**
  - **Recall**
  - **ROC-AUC**
  (а не лише Accuracy через дисбаланс класів)
